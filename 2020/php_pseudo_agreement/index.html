php伪协议总结

PHP伪协议是什么?

php 伪协议是 ctf 中常用的工具。如何理解伪协议？可以把伪协议当作是一个真实文件。这个文件的文件名为伪协议名，文件内容为各个伪协议的内容。

PHP伪协议的种类

PHP伪协议事实上就是其支持的协议与封装协议(12种)
a. file:// — 访问本地文件系统
b. http:// — 访问 HTTP(s) 网址
c. ftp:// — 访问 FTP(s) URLs
d. php:// — 访问各个输入/输出流（I/O streams）
e. zlib:// — 压缩流
f. data:// — 数据（RFC 2397）
g. glob:// — 查找匹配的文件路径模式
h. phar:// — PHP 归档
i. ssh2:// — Secure Shell 2
j. rar:// — RAR
k. ogg:// — 音频流
l. expect:// — 处理交互式的流

与伪协议息息相关的文件包含函数

伪协议的使用是往往离不开文件包含函数：

文件包含函数函数功能
include()包含并运行指定文件。
require()
include_once()只包含一次
require_once()
highlight_file()打印输出或返回文件中语法高亮版本的代码
show_source()ighlight_file的别名
readfile()读取文件并写入到输出缓冲
file_get_contents()将整个文件读入一个字符串
fopen()打开文件或者 URL
file()把整个文件读入一个数组中

伪协议的使用条件

在php.ini里有两个重要的参数allow_url_fopen和allow_url_include
allow_url_fopen:默认值是ON，允许url里的封装协议访问文件
allow_url_include:默认值是OFF,不允许包含url里的封装协议包含文件

PHP://filter

经常使用的伪协议，一般用于任意文件读取，有时也可以用于getshell;
在双OFF的情况下也可以使用.

名称描述备注
resource=<要过滤的数据流>指定了你要筛选过滤的数据流。必选
read=<读链的筛选列表>可以设定一个或多个过滤器名称，以管道符（）分隔。
write=<写链的筛选列表>可以设定一个或多个过滤器名称，以管道符（）分隔。
<；两个链的筛选列表>任何没有以 read= 或 write= 作前缀 的筛选器列表会视情况应用于读或写链。

这个协议常用来读取服务器文件的内容，输出base64编码的文件内容。
用法：php://filter/read=convert.base64-encode/resource=[文件名]
例题：http://123.206.87.240:8005/post/index.php

http://、https://

URL 形式，允许通过 HTTP 1.0 的 GET方法，以只读访问文件或资源，通常用于远程包含。受 allow_url_fopen 限制；
用法： ?file=http://127.0.0.1/phpinfo.txt
不常使用

file://

本地文件传输协议：file://[文件的绝对路径和文件名]
在双OFF的时候也可以用
file协议主要用于访问本地计算机中的文件，好比通过Windows的资源管理器中打开文件或者通过右键单击‘打开’一样。
用法: 访问index.php?file=file://d:/flag.txt，发现读取了d:/flag.txt的内容
不常使用

phar://

PHP 归档，常常跟文件包含，文件上传结合着考察。
说通俗点就是php解压缩包的一个函数，解压的压缩包与后缀无关。
用法: phar://[压缩文件路径]/[压缩文件内的子文件名]

例如脚本文件为1.php,里面写入打包成1.zip,然后再改名为1.jpg,然后上传到服务器
访问index.php?file=phar://1.jpg/1.php,发现已经成功执行了phpinfo()命令
若1.php内容为一句话，也可以使用工具进行连接

zip://,bzip2://, zlib://

都属于压缩流，可以访问压缩文件中的子文件。
在双Off情况下可以使用
格式：zip://[压缩文件绝对路径]%23[压缩文件内的子文件名]（#编码为%23）
compress.bzip2://file.bz2(相对路径和绝对路径都可以，必须为bz2格式的压缩文件)
compress.zlib://file.gz(相对路径和绝对路径都可以)
