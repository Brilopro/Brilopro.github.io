CentOS8搭建H1ve平台

1、 安装docker

1.1安装所需软件包

sudo yum install -y yum-utils device-mapper-persistent-data lvm2 bind-utils

1.2设置yum源

Sudoyum-config-manager --add-repohttp://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

1.3安装docker

Sudo yum install docker-ce
遇到问题

根据提示在命令结尾加上–allowerasing或–nobest后再次执行即可
yum install -y docker-ce –nobest

1.4启动并加入开机启动

sudo systemctl start docker #或sudo service docker start
sudo systemctl enable docker

1.5验证是否安装成功

docker version

2、 安装docker-compose

1. 安装epel-release

yum -y install epel-release

2. 安装python-pip(centOS 8 自带pip3 所以不用安装)

yum -y install python-pip

3. 升级pip

sudo pip install --upgrade pip
pip3使用下面的命令
sudo pip3 install --upgrade pip
如果安装失败就多试几次

4. 安装docker-compose

pip3 install docker-compose
出现错误

出现错误根据提示改用命令
sudo python3 -m pip install docker-compose

  1. 验证docker-compose是否安装成功
    docker-compose version

3、 安装与启动H1ve

1.下载H1ve
第一种方法：
下载git
sudo yum install git

使用git下载H1ve
Git clone https://github.com/D0g3-Lab/H1ve.git
第二种方法访问网站https://github.com/D0g3-Lab/H1ve 下载源码并解压

下载完成后进入下载文件夹将压缩包提取到主目录

2.配置docker源为国内源
这个主要是因为当拉取镜像时，由于远端仓库的服务器是在国外，国内用户拉取镜像的速度特别慢，或很可能都访问不了，产生报错。
建议使用阿里云的镜像加速器比较快，或者使用中科院的
ustc(中国科学技术大学)：
https://docker.mirrors.ustc.edu.cn

3.进入H1ve目录 启动H1ve
进入H1ve
cd H1ve

启动H1ve
sudo docker-compose -f single.yml up &
在执行是可能会遇到错误

解决办法:
sudo ln -s /usr/local/bin/docker-compose /usr/bin/docker-compose
重新启动H1ve
sudo docker-compose -f single.yml up &
中间遇到几次问题
错误详情：
docker.errors.DockerException: Error while fetching server API version: ('Connection aborted.', PermissionError(13, 'Permission denied'))
出现错误的原因是因为某些权限的原因，将当前用户加入docker用户组就行

  1. 创建docker用户组.
    sudo groupadd docker
  2. 将当前用户加入docker用户组.
    sudo usermod -aG docker $USER
  3. 注销并从新登录用户

再次报错,错误详情:
ERROR:h1ve_main_1 exited with code 1
删除H1ve目录中的隐藏文件夹.data,关闭H1ve 修改配置文件single.yml

docker-compose启动容器时出现报错
Creating network "soft_default" with the default driver
解决方法如下：

Issue Description (问题描述)

解决方法，同样是把用户加入docker用户组

  1. 创建docker用户组.
    sudo groupadd docker
  2. 将当前用户加入docker用户组.
    sudo usermod -aG docker $USER
  3. 注销并从新登录用户

再次报错:

网上搜索了一下，也有人遇到过类似情况，关闭了服务器防火墙就OK了

注意每次出现错误，都需要先down掉H1ve再修复然后再up H1ve

访问本机ip:8000 H1ve至此搭建成功
